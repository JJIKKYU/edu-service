## Overview
iOS(Swift/ObjC)에서 KMP(TCK - ThinQ CoreKit)로의 마이그레이션 진행 현황을 파일/함수 단위로 추적하는 대시보드. 소스 파일을 업로드하면 함수·클래스·프로토콜을 자동 파싱하고, 각 함수의 TCK 전환 완료 여부를 체크하며 전체 진행률을 한눈에 확인한다.

## 시나리오

### 1. 빈 상태 안내
[상황] 등록된 파일이 없는 상태에서 대시보드에 진입하면
[동작] 빈 상태 안내 문구와 파일 업로드 버튼이 표시된다

성공 기준:
- [ ] 등록 파일 0개 -> "등록된 파일이 없습니다" 안내 문구 표시
- [ ] 등록 파일 0개 -> "파일 업로드" 버튼 표시
- [ ] 등록 파일 0개 -> 대시보드 요약 영역에 파일 수 0, 함수 수 0, 완료율 0% 표시

### 2. Swift 파일 업로드 및 파싱
[상황] 사용자가 .swift 파일을 업로드하면
[동작] 시스템이 파일을 파싱하여 함수·클래스·프로토콜 계층 구조를 추출하고 파일 카드를 표시한다

성공 기준:
- [ ] `HomeViewModel.swift` (func loadData, func refresh, class HomeViewModel) 업로드 -> 파일 카드에 "HomeViewModel.swift" 표시, 함수 2개 표시
- [ ] 파일 카드에 진행률 프로그레스 바 0% 표시
- [ ] 대시보드 요약에 파일 수 1, 함수 수 2, 완료율 0% 반영

### 3. ObjC 파일 업로드 및 파싱
[상황] 사용자가 .m 또는 .h 파일을 업로드하면
[동작] 시스템이 파일을 파싱하여 메서드·클래스·프로토콜 계층 구조를 추출하고 파일 카드를 표시한다

성공 기준:
- [ ] `NetworkManager.m` (- (void)fetchData, - (void)cancelRequest, @interface NetworkManager) 업로드 -> 파일 카드에 "NetworkManager.m" 표시, 함수 2개 표시
- [ ] 파일 카드에 진행률 프로그레스 바 0% 표시
- [ ] 대시보드 요약에 파일 수·함수 수·완료율 누적 반영

### 4. 파일 카드 확장하여 함수 목록 확인
[상황] 파일 카드가 접힌 상태에서 카드를 클릭하면
[동작] 카드가 확장되어 해당 파일의 함수 목록이 클래스/프로토콜별로 그룹화되어 표시된다

성공 기준:
- [ ] "HomeViewModel.swift" 카드 클릭 -> "HomeViewModel" 클래스 아래 "loadData", "refresh" 함수 목록 표시
- [ ] 각 함수 옆에 체크박스 표시 (미체크 상태)

### 5. 함수 TCK 전환 완료 체크
[상황] 확장된 파일 카드에서 함수의 체크박스를 클릭하면
[동작] 해당 함수가 TCK 전환 완료로 표시되고, 파일 진행률과 대시보드 요약이 갱신된다

성공 기준:
- [ ] 함수 2개 중 "loadData" 체크 -> 파일 프로그레스 바 50% 표시
- [ ] 함수 2개 중 "loadData" 체크 -> 대시보드 완료율 갱신 (전체 함수 대비 완료 함수 비율)
- [ ] 체크된 함수를 다시 클릭 -> 체크 해제, 진행률 원복

### 6. 대시보드 집계 요약
[상황] 파일이 등록된 상태에서 대시보드를 확인하면
[동작] 전체 파일 수, 전체 함수 수, 전체 완료율이 요약 영역에 표시된다

성공 기준:
- [ ] 파일 2개(함수 2개 + 함수 3개 = 총 5개), 완료 2개 -> 파일 수 2, 함수 수 5, 완료율 40% 표시
- [ ] 완료율은 소수점 없이 반올림하여 표시 (예: 40%)

### 7. 중복 파일 업로드 거부
[상황] 이미 등록된 파일과 같은 이름의 파일을 업로드하면
[동작] 중복 안내 메시지를 표시하고 업로드를 거부한다

성공 기준:
- [ ] "HomeViewModel.swift" 등록 후 같은 이름 파일 업로드 -> "이미 등록된 파일입니다" 메시지 표시
- [ ] 기존 파일 데이터 유지, 파일 수 변화 없음

### 8. 지원하지 않는 파일 형식 업로드 거부
[상황] .swift/.m/.h가 아닌 파일을 업로드하면
[동작] 지원하지 않는 형식임을 안내하는 메시지를 표시하고 업로드를 거부한다

성공 기준:
- [ ] `utils.py` 업로드 -> "지원하지 않는 파일 형식입니다" 메시지 표시
- [ ] 파일 수 변화 없음

### 9. 파일 카드 접기 (토글)
[상황] 파일 카드가 확장된 상태에서 카드를 다시 클릭하면
[동작] 카드가 접혀 함수 목록이 숨겨진다

성공 기준:
- [ ] 확장된 "HomeViewModel.swift" 카드 클릭 -> 함수 목록 숨김

### 10. ObjC 헤더(.h) 파일 업로드 및 파싱
[상황] 사용자가 .h 파일을 업로드하면
[동작] 시스템이 파일을 파싱하여 메서드 선언·클래스·프로토콜 계층 구조를 추출하고 파일 카드를 표시한다

성공 기준:
- [ ] `NetworkManager.h` (@interface NetworkManager, - (void)fetchData;, - (void)cancelRequest;) 업로드 -> 파일 카드에 "NetworkManager.h" 표시, 함수 2개 표시
- [ ] 파일 카드에 진행률 프로그레스 바 0% 표시

### 11. 파일 내 모든 함수 체크 완료
[상황] 확장된 파일 카드에서 모든 함수의 체크박스를 클릭하면
[동작] 파일 진행률이 100%로 갱신되고 대시보드 완료율이 100%로 갱신된다

성공 기준:
- [ ] 함수 2개 모두 체크 -> 파일 프로그레스 바 100% 표시
- [ ] 대시보드 완료율 100% 표시

## 범위

### 포함
- .swift, .m, .h 파일 업로드 및 파싱 (함수 + 클래스/프로토콜 계층)
- 파일별 진행률 프로그레스 바
- 전체 요약 대시보드 (파일 수, 함수 수, 완료율)
- 함수별 TCK 전환 완료 체크/언체크
- Prisma + SQLite로 데이터 저장 (Next.js Route Handlers)

### 제외
- 드래그앤드롭 파일 업로드: 첫 버전에서는 클릭 업로드만 지원하여 구현 복잡도를 줄인다
- 파일 삭제: 삭제 시 체크 데이터 처리 정책이 미확정이므로 제외한다
- 함수 본문 파싱: 시그니처(이름, 파라미터)만 추출하며 본문 분석은 범위 밖이다
- 다중 사용자 동기화: 단일 사용자 환경을 전제한다

## 전제 조건
- Next.js App Router 환경
- Prisma + SQLite 설정 완료
- 업로드 파일은 UTF-8 인코딩된 텍스트 파일

## 미결정 사항
- 파싱 실패 시 처리 방식 (에러 표시 vs 부분 결과 표시)
- 파일 re-upload 시 기존 체크 상태 병합 여부
- 파일 크기 제한 (현재 미설정)
- 함수별 메모/코멘트 기능 추가 여부
